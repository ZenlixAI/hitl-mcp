# hitl-mcp

Question-only HITL MCP server for agent workflows.

## Public Model

- The public unit is `question`.
- Multiple pending questions can exist in one caller scope.
- Answers can be submitted partially and accumulated server-side.
- Internal grouping may still exist in storage, but it is not part of MCP tools or HTTP APIs.

Caller scope is:

- `agent_identity`
- `agent_session_id` from `x-agent-session-id` by default

## MCP Tools

- `hitl_ask`
- `hitl_wait`
- `hitl_get_pending_questions`
- `hitl_submit_answers`
- `hitl_cancel_questions`
- `hitl_get_question`

## HTTP API

- `POST /api/v1/questions`
- `GET /api/v1/questions/pending`
- `POST /api/v1/questions/answers`
- `POST /api/v1/questions/cancel`
- `GET /api/v1/questions/:question_id`

## Wait Modes

Configure with `HITL_WAIT_MODE`:

- `terminal_only`: `wait` returns only when the caller scope has no pending questions left.
- `progressive`: `wait` returns after each question-state change. Call `wait` again to continue observing.

Default: `terminal_only`

## Partial Submission

`POST /api/v1/questions/answers` and `hitl_submit_answers` accept:

- `answers`: subset of pending questions to answer
- `skipped_question_ids`: subset of optional pending questions to skip
- `idempotency_key`: optional

Each submit persists incremental progress. A full batch completion is not required.

`question_id` is generated by the MCP server. Do not send it in `hitl_ask` or `POST /api/v1/questions`.

Default bind address: `0.0.0.0:4000`

## Example

Create questions:

```json
{
  "title": "Release Decision",
  "questions": [
    {
      "type": "single_choice",
      "title": "Can we start canary deployment?",
      "options": [
        { "value": "yes", "label": "Yes" },
        { "value": "no", "label": "No" }
      ]
    },
    {
      "type": "text",
      "title": "Anything to note?",
      "required": false
    }
  ]
}
```

The response returns server-generated `question_id` values. Use those IDs in later `submit/cancel/get` calls.

Submit part of the answers:

```bash
curl -X POST "http://localhost:4000/api/v1/questions/answers" \
  -H "Content-Type: application/json" \
  -H "x-agent-identity: api_key:test-agent" \
  -H "x-agent-session-id: session-123" \
  -d '{
    "answers": {
      "q_canary": { "value": "yes" }
    }
  }'
```

## Configuration

- `PORT`
- `MCP_URL`
- `HITL_PENDING_MAX_WAIT_SECONDS`
- `HITL_WAIT_MODE=terminal_only|progressive`
- `HITL_STORAGE=memory|redis`
- `HITL_REDIS_URL`
- `HITL_REDIS_PREFIX`
- `HITL_TTL_SECONDS`
- `HITL_ANSWERED_RETENTION_SECONDS`
- `HITL_API_KEY`
- `HITL_AGENT_AUTH_MODE`
- `HITL_AGENT_SESSION_HEADER`
- `HITL_CREATE_CONFLICT_POLICY`
- `HITL_SERVER_NAME`
- `HITL_SERVER_VERSION`
- `HITL_HTTP_HOST`
- `HITL_HTTP_API_PREFIX`
- `HITL_LOG_LEVEL`
- `HITL_ENABLE_METRICS`
